# VoiceToText Bot — Концепция

## 1. Общее описание

Telegram-бот для распознавания речи из аудио/видео файлов с преобразованием в форматированный текст.

**Пайплайн обработки:**
```
Telegram (файл) → FFmpeg (извлечение аудио) → AssemblyAI (STT + диаризация) → OpenRouter LLM (форматирование + summary) → Telegram (результат)
```

Вся тяжёлая обработка происходит в облаке. На сервере выполняется только приём файлов и извлечение аудио через FFmpeg.

---

## 2. Выбранный стек технологий

| Компонент | Технология | Назначение |
|-----------|-----------|------------|
| **Бот** | Python 3.11+ / aiogram 3 | Telegram Bot Framework (async) |
| **STT + Диаризация** | AssemblyAI API | Распознавание речи + разделение по спикерам |
| **Форматирование** | OpenRouter API (бесплатные модели) | LLM для форматирования текста и summary |
| **Извлечение аудио** | FFmpeg + pydub | Конвертация видео → аудио |
| **БД** | SQLite + aiosqlite | Пользователи, история, настройки |
| **Деплой** | Docker + docker-compose | Контейнеризация |

### Почему этот стек

- **AssemblyAI** — лучшая диаризация «из коробки», минимум кода. Бесплатный tier: $50 credits ≈ 185 часов аудио
- **OpenRouter (бесплатные модели)** — Gemma, Llama, Mistral бесплатно для форматирования текста. $10 баланс не тратится на STT
- **Нагрузка на сервер минимальна** — Intel N95 с 8 ГБ RAM легко справится (только бот + FFmpeg)
- **Скорость** — 10 мин аудио ≈ 30 сек обработки

### Оценка стоимости

| Операция | Стоимость |
|----------|----------|
| STT + диаризация (AssemblyAI) | $0.15-0.37/час аудио |
| Форматирование (OpenRouter, бесплатные модели) | $0.00 |
| **10-мин файл** | **~$0.03-0.06** |
| **Месяц (3-4 раза/неделю по 10 мин)** | **~$0.50-1.00** |
| **Бесплатных кредитов ($50) хватит на** | **~185 часов** |

## 3. AssemblyAI — регистрация и настройка

### Что такое AssemblyAI

AssemblyAI — специализированный облачный сервис для распознавания речи. Ключевые возможности:
- Speech-to-Text (STT) с высокой точностью
- **Speaker Diarization** — автоматическое разделение по спикерам
- Поддержка 99+ языков (включая русский и английский)
- Автоматическая пунктуация и форматирование

### Бесплатный тарифный план

| Параметр | Значение |
|----------|---------|
| Бесплатные кредиты | **$50** (единоразово при регистрации) |
| Примерный объём | ~185 часов аудио (базовая модель) |
| Привязка карты | **Не требуется** |
| Ограничения | До 5 одновременных запросов |
| Срок действия | Бессрочно (пока не израсходуются) |

### Регистрация (пошагово)

1. Перейти на [assemblyai.com](https://www.assemblyai.com)
2. Нажать **«Get Started Free»** (или «Sign Up»)
3. Зарегистрироваться через **Google-аккаунт** или по **email**
   - При регистрации по email — на почту придёт magic-link для входа
4. После входа откроется **Dashboard**
5. Перейти в раздел **«API Keys»**
6. Нажать **«+ Create new API key»**
7. Дать ключу название (например, `voicetotext-bot`)
8. **Скопировать ключ** и сохранить в безопасное место

> ⚠️ API-ключ показывается только один раз. Храните его в `.env` файле, никогда не коммитьте в Git.

## 4. Пользователи и авторизация

### Роли
| Роль | Описание |
|------|----------|
| **Админ** | Полный доступ. Управляет пользователями, видит статистику |
| **Пользователь** | Отправляет файлы и получает расшифровки |
| **Неавторизованный** | Видит приветствие и может отправить запрос на доступ |

### Процесс авторизации
1. Новый пользователь нажимает `/start` → видит приветствие с описанием бота
2. Нажимает кнопку **«Запросить доступ»** → бот отправляет админу уведомление (имя, username, ID)
3. Админ нажимает **✅ Одобрить** или **❌ Отклонить** (inline-кнопки)
4. Пользователь получает уведомление о решении

### Команды админа
| Команда | Описание |
|---------|----------|
| `/users` | Список всех пользователей и их статусы |
| `/approve <id>` | Одобрить доступ |
| `/revoke <id>` | Отозвать доступ |
| `/stats` | Статистика (кол-во расшифровок, объём аудио, расход кредитов) |

## 5. Поддерживаемые форматы

### Нативные Telegram-форматы
- 🎤 Голосовые сообщения (voice messages)
- 🎥 Видеосообщения (video notes / кружочки)
- 📎 Пересланные голосовые и видео

### Загружаемые файлы
| Тип | Форматы |
|-----|---------|
| Аудио | MP3, WAV, OGG, FLAC, M4A, AAC, WMA |
| Видео | MP4, AVI, MKV, MOV, WEBM |

> Из видео автоматически извлекается аудиодорожка через FFmpeg.

### Ограничения
- Telegram Bot API: максимум **20 МБ** на файл
- Максимальная длительность: **2 часа** (настраиваемый лимит)

## 6. Рабочий процесс и статусы

### Workflow

```
Пользователь              Бот                    Облако
    │                      │                       │
    │── отправляет файл ──►│                       │
    │◄── ⏳ Принято ───────│                       │
    │                      │── извлечь аудио ─────►│
    │◄── 🔄 Подготовка ───│                       │
    │                      │── отправить в API ───►│
    │◄── 🎙 Распознаю ────│        AssemblyAI     │
    │                      │◄── текст + спикеры ──│
    │◄── 👥 Спикеры ──────│                       │
    │                      │── отправить в LLM ──►│
    │◄── ✍️ Форматирую ───│        OpenRouter     │
    │                      │◄── готовый текст ────│
    │◄── ✅ Результат ────│                       │
```

### Статусы обработки

Бот отправляет **отдельные сообщения** для каждого этапа (хронология видна в чате):

| Этап | Сообщение | Описание |
|------|----------|----------|
| Принято | ⏳ Файл получен, ставлю в очередь... | Файл скачан |
| Подготовка | 🔄 Подготавливаю аудио... | Конвертация, извлечение дорожки |
| Распознавание | 🎙 Распознаю речь... | STT обработка в AssemblyAI |
| Диаризация | 👥 Определяю спикеров... | Разделение по говорящим |
| Форматирование | ✍️ Форматирую текст... | LLM постобработка |
| Готово | ✅ Расшифровка готова! | Результат отправлен |
| Ошибка | ❌ Ошибка: [описание] | Не удалось обработать |

**Для длинных файлов (>10 мин):** бот периодически отправляет heartbeat-сообщения:
```
🎙 Распознаю речь... обработано 5 из 12 минут
🎙 Распознаю речь... обработано 10 из 12 минут
```

## 7. Формат результата

### Пример вывода
```
📝 Расшифровка аудио
━━━━━━━━━━━━━━━━━━
📁 Файл: meeting_2024_03_01.mp3
⏱ Длительность: 12:34
🗣 Спикеров: 3
🌐 Язык: Русский
━━━━━━━━━━━━━━━━━━

🔵 Спикер 1 [00:00:05]
Добрый день, коллеги. Давайте начнём
совещание по проекту.

🟢 Спикер 2 [00:00:12]
Здравствуйте! У меня есть обновления
по фронтенду.

🔴 Спикер 3 [00:00:25]
Отлично, давайте начнём с этого.

...

━━━━━━━━━━━━━━━━━━
📊 Краткое содержание:
• Обсудили прогресс по фронтенду
• Назначили дедлайн на пятницу
• Решили добавить новую фичу в бэклог
```

### Правила форматирования
- Каждый спикер обозначен **цветным эмодзи** (🔵🟢🔴🟡🟣🟤)
- Указаны **временные метки** начала реплики
- **Абзацы** разделены по смысловым блокам
- В конце — **краткое содержание** (summary) от LLM
- Для длинных расшифровок — отправка **текстовым файлом** (.txt)
- Лимит Telegram: 4096 символов → длинные тексты разбиваются на несколько сообщений

## 8. Дополнительные функции

### MVP (первая версия)
- 🌐 **Автоопределение языка** — AssemblyAI сам определяет язык
- 📄 **Экспорт в файл** — для длинных расшифровок .txt файлом
- 🔁 **Повторная обработка** — кнопка «Повторить» при ошибке
- 📊 **Краткое содержание** — автоматическое резюме через LLM

### v2 (улучшения)
- 🏷 **Именование спикеров** — назначить имена после расшифровки
- 🔍 **Выбор языка вручную** — если автоопределение ошиблось
- 📋 **История расшифровок** — `/history` с последними расшифровками
- ⚙️ **Настройки пользователя** — предпочитаемый язык, формат вывода

### v3 (перспективные)
- 📌 **Ключевые моменты** — LLM выделяет решения и action items
- 🔗 **Интеграция** — отправка результатов в Google Docs / Notion
- 🎯 **Словарь терминов** — пользовательский словарь
